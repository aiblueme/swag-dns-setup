01
Domain Delegation
Transfer DNS authority from Namecheap to Cloudflare
  • Log into Namecheap, select your domain
  • Set Nameservers to "Custom DNS"
  • Add Cloudflare Nameservers (found in Cloudflare Dashboard → your domain → Overview)
  • Wait for "Active" status in Cloudflare (can take up to 24h)
02
DNS Records & API Token
Configure wildcard DNS entry and generate SWAG API token
  • Create A record: @ → YOUR_VPS_IP (Proxy: OFF / grey cloud)
  • Create CNAME record: * → yourdomain.com (Proxy: OFF / grey cloud)
  • Go to Cloudflare → My Profile → API Tokens → Create Token
  • Use "Edit zone DNS" template
  • Set permissions: Zone.DNS:Edit + Zone.Zone:Read
  • Set scope: Include → yourdomain.com
  • Save token securely (shown only once)
03
Deploy SWAG Container
Set up the SWAG Docker container on your VPS
  • SSH into YOUR_VPS_IP
  • Run: mkdir -p ~/docker/swag && cd ~/docker/swag
  • Create docker-compose.yml using the code block — edit domain and email to match yours
  • Run: docker compose up -d (initializes config folder structure)
services: swag: image: lscr.io/linuxserver/swag:latest container_name: swag cap_add: - NET_ADMIN environment: - PUID=1000 - PGID=1000 - TZ=Etc/UTC - URL=yourdomain.com - SUBDOMAINS=wildcard - VALIDATION=dns - DNSPLUGIN=cloudflare - EMAIL=admin@yourdomain.com volumes: - ./config:/config ports: - 443:443 - 80:80 restart: unless-stopped
04
Wildcard SSL Auth
Link SWAG to Cloudflare via API token for ACME DNS challenge
  • Stop container: docker compose stop
  • Edit file: ~/docker/swag/config/dns-conf/cloudflare.ini
  • Comment out all lines except dns_cloudflare_api_token
  • Set: dns_cloudflare_api_token = YOUR_CLOUDFLARE_API_TOKEN
  • Start container: docker compose up -d
  • Watch logs: docker logs -f swag
  • Confirm "Server Ready" appears in logs
# /config/dns-conf/cloudflare.ini # dns_cloudflare_email = admin@yourdomain.com ← keep commented out dns_cloudflare_api_token = YOUR_CLOUDFLARE_API_TOKEN
05
Service Routing
Enable wildcard subdomains for internal Docker apps
  • Navigate to: ~/docker/swag/config/nginx/proxy-confs/
  • Find a .subdomain.conf.sample file for your app (e.g. heimdall.subdomain.conf.sample)
  • Copy it: cp app.subdomain.conf.sample app.subdomain.conf
  • Edit the file: set proxy_pass to your container's internal Docker name/IP + port
  • Restart SWAG: docker compose restart swag
  • Test: open https://app.yourdomain.com in browser
